# Shadow Economy Prediction Model

This repository implements a supervised machine learning framework to predict shadow economy levels (as a percentage of GDP) in Eastern European countries.  
Multiple algorithms are applied and compared, including XGBoost, CatBoost, Support Vector Regression (SVR), Random Forest, and Elastic Net.

## Repository Structure

### Data
- `data/`  
  Main data directory.

- `data/cause_variables/`  
  Raw datasets for explanatory variables sourced from:
  - World Bank
  - Worldwide Governance Indicators (WGI)
  - International Monetary Fund (IMF)
  - Transparency International
  - Other international institutions

- `data/target_variable/`  
  Target variable dataset:
  - `shadow_economy%.xlsx`

- `data/master_panel_.xlsx`  
  Master panel dataset created by combining all raw explanatory variables using `dataset.py`.  
  This dataset is subsequently filtered into `filtered_data_2000.xlsx`.

- `data/df_tree.xlsx`  
  Final dataset used for tree-based and kernel-based models:
  - XGBoost
  - CatBoost
  - SVR
  - Random Forest  

  The dataset includes transformed variables,target variable, missing values, and missing-value indicators.

- `data/df_elastic_net.xlsx`  
  Final dataset used for Elastic Net models.  
  All variables are scaled and missing values are removed, as required by the algorithm.

- `figures/`  
  Contains figures and visual outputs generated by the analysis notebooks.

---

### Notebooks and Scripts
- `dataset.py`  
  Script that merges raw explanatory variables into the master panel dataset.

- `dataset_cleaned.ipynb`  
  Performs data cleaning and preprocessing, including:
  - handling missing values  
  - feature selection  
  - data transformation and normalization  
  - creation of two final datasets:
    - `df_tree.xlsx` (for tree-based and SVR models)
    - `df_elastic_net.xlsx` (for Elastic Net models)

- `machine_learning_test.ipynb`  
  Trains and evaluates the machine learning models:
  - XGBoost
  - CatBoost
  - SVR
  - Random Forest
  - Elastic Net  

  Models are evaluated across three specifications:
  - **Model I:** Level models with lagged shadow economy
  - **Model II:** One-year change models
  - **Model III:** Three-year change models  

  Evaluation periods:
  - In-sample: 2000–2015  
  - Out-of-sample: 2016–2023  
  - Hold-out test: 2024  

  The notebook also includes scenario analysis and feature importance evaluation.
  
## How to Run the Code

### 1. Environment Setup

The project is written in Python and requires the following:

- **Python 3.8 or higher** 
- **Jupyter Notebook** (or JupyterLab)

Note: for Catboost was used Python 3.11.14. Newer versions may cause compatibility issues.

### 2. Required Libraries

The following Python libraries are required:

**Core Data Processing:**
- `pandas` - Data manipulation and analysis
- `numpy` - Numerical computing

**Machine Learning:**
- `scikit-learn` - Machine learning algorithms and utilities
- `xgboost` - Gradient boosting framework
- `catboost` - Gradient boosting library with categorical features support

**Visualization:**
- `matplotlib` - Plotting and visualization
- `seaborn` - Statistical data visualization

**Statistical Analysis:**
- `scipy` - Scientific computing and statistics
- `statsmodels` - Statistical modeling

**File Handling:**
- `openpyxl` - Reading and writing Excel files (.xlsx)

### 2. Data Preparation

Before running the machine learning models, ensure the data files are in the correct location:

- **Target variable:** `data/target variable/shadow_economy%.xlsx`
- **Preprocessed datasets:** The notebooks will generate and use:
  - `data/df_tree_train_all.csv`
  - `data/df_tree_test_all.csv`
  - `data/df_elastic_train_all.csv`
  - `data/df_elastic_test_all.csv`

### 3. Running the Notebooks

#### Step 1: Data Cleaning and Preparation

Run the data preparation notebook first:
```bash
jupyter notebook dataset_cleaned.ipynb
```

These notebooks will:
- Load and merge data from various sources
- Create train/test splits
- Generate preprocessed datasets for tree-based models and Elastic Net models
- Save processed data to CSV files

#### Step 2: Run the Main Analysis

Open and run the main machine learning notebook:
```bash
jupyter notebook machine_learning_test.ipynb
```

This notebook contains:
- **Model 1:** Level models with Shadow_Lag1 feature
- **Model 2:** Change models (1-year difference)
- **Model 3:** 3-year change models
- Algorithms: XGBoost, CatBoost, SVR, Random Forest, Elastic Net
- Hyperparameter tuning using GridSearchCV with TimeSeriesSplit
- Model evaluation and visualization
- Feature importance analysis

**Execution Order:**
1. Cell 0: Import all required libraries
2. Cell 1: Load preprocessed datasets
3. Cell 2: Load target variable data
4. Cells 3-11: Data preparation and feature engineering
5. Cells 12+: Individual model training and evaluation

### Running the Data Processing Script (Optional)

If you need to regenerate the dataset from raw data files:
```bash
python dataset.py
```

**Note:** Ensure all raw data files are in the `data/cause variables/` directory.

### 7. Expected Outputs

After running the notebooks, you should see:

- **Processed datasets** saved in `data/` directory:
  - `df_tree.xlsx`
  - `df_elastic_net.xlsx`
  - Various CSV files for train/test splits

- **Figures and visualizations** in `figures/` directory:
  - Model performance comparisons
  - Feature importance plots
  - Prediction visualizations

- **Model results** displayed in notebook outputs:
  - RMSE, MAE, R² metrics
  - Sign accuracy
  - Improvement percentages over baseline models
---

## Notes

- The notebooks use `TimeSeriesSplit` for cross-validation to maintain temporal order
- Models use expanding-window walk-forward validation (training on all data up to test year)
- Missing values are handled differently for each algorithm (CatBoost handles NaNs natively, others use imputation)
- All models are evaluated on test years 2016-2023.
